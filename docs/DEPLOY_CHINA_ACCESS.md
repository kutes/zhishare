# 让网站在中国大陆能打开 —— 小白版分步教程

> 目标:给现在的 Vercel 网站接一个**自己的域名 + Cloudflare(免费)**,让大陆用户能访问。
> 原理:`zhishare.vercel.app` 这个域名在大陆被封;换成自己的域名走 Cloudflare(和参考站 best.xiaohu.ai 同款做法),大陆就能打开(偏慢但可用)。
> 花费:只有域名钱,约 **¥60–90/年**。不用备案,代码一行不改。

---

## 你需要准备

- 一个能用**支付宝/微信**付款的账号(买域名用)
- 一个邮箱(注册 Cloudflare 用)
- 你现在的 Vercel 账号(网站已经部署在上面)

全程约 30–60 分钟,其中大部分时间在**等 DNS 生效**(可以去干别的)。

---

## 第 1 步:买一个域名

推荐在 **阿里云** 或 **腾讯云** 买(中文界面、支付宝付款,对小白最友好)。

1. 打开 阿里云域名(wanwang.aliyun.com)或 腾讯云域名(dnspod.cloud.tencent.com)。
2. 搜一个你想要的名字,选 **`.com`** 后缀(最通用)。例如 `zhishare.com`。
3. 下单付款。**实名认证**按提示做(买域名需要,和备案是两回事,填一下就行)。

> 说明:这里**不需要备案**。备案只有在用"中国境内的服务器"时才要,我们用的是海外的 Vercel + Cloudflare,所以跳过。

买完你就拥有一个域名了,记住它(下面叫「你的域名」)。

---

## 第 2 步:把域名加进 Cloudflare(免费)

1. 打开 **cloudflare.com**,用邮箱注册一个账号(免费)。
2. 登录后点 **Add a site / 添加站点**,输入「你的域名」(只填主域名,如 `zhishare.com`,不要带 www)。
3. 套餐选 **Free($0)**,继续。
4. Cloudflare 会自动扫描现有 DNS 记录 —— 直接下一步即可(我们后面会自己加)。
5. Cloudflare 会给你**两个 Nameserver(域名服务器)地址**,长得像:
   ```
   xxx.ns.cloudflare.com
   yyy.ns.cloudflare.com
   ```
   **把这两个记下来。**

### 回到买域名的地方,改 Nameserver

1. 回到 阿里云/腾讯云 的「域名控制台」→ 找到你的域名 → **管理 / DNS 修改 / 修改 DNS 服务器**。
2. 把默认的 DNS 服务器**替换成 Cloudflare 给的那两个地址**,保存。
3. 改完回到 Cloudflare 点 **Done / 完成**。

> 之后 Cloudflare 会检查、生效。**可能几分钟到几小时**,生效后 Cloudflare 会发邮件通知你「Active(已激活)」。这一步只能等,不用一直盯着。

---

## 第 3 步:在 Vercel 绑定这个域名

1. 打开 **vercel.com** → 进入你的 `zhishare` 项目 → **Settings → Domains**。
2. 输入「你的域名」(比如 `zhishare.com`),点 **Add**。
3. 再加一个带 www 的:`www.你的域名`(比如 `www.zhishare.com`),点 Add。
4. Vercel 会显示需要配置的 DNS 记录,通常是:
   - 主域名(apex):**A 记录 → `76.76.21.21`**
   - www:**CNAME 记录 → `cname.vercel-dns.com`**

   **记下这两条**(Vercel 页面上会写清楚,以它显示的为准)。

---

## 第 4 步:在 Cloudflare 里加这两条 DNS(最关键)

回到 Cloudflare → 选你的域名 → 左边 **DNS → Records → Add record**,加两条:

| 类型 | 名称(Name) | 内容(Content) | 代理状态 |
|---|---|---|---|
| A | `@` | `76.76.21.21` | **橙色云朵(Proxied)** |
| CNAME | `www` | `cname.vercel-dns.com` | **橙色云朵(Proxied)** |

> ⚠️ **两条都必须是「橙色云朵」(Proxied / 已代理)**,不能是灰色。
> 橙色 = 流量走 Cloudflare 的网络(大陆能连的关键);灰色 = 直连 Vercel = 又被封回去了。

### 再改一个开关,否则会打不开(必看)

Cloudflare → **SSL/TLS → Overview(概述)** → 加密模式选 **Full(完全)**。

> 这是最常见的坑:如果这里是默认的「Flexible(灵活)」,网站会**无限跳转 / 打不开(ERR_TOO_MANY_REDIRECTS)**。一定要改成 **Full**。

---

## 第 5 步:验证

1. 等 Cloudflare 状态变 **Active**、Vercel 的 Domains 里两个域名都打勾(Valid Configuration)。
2. 用手机/电脑打开 **`https://你的域名`**,能看到网站就成功了。
3. 想确认大陆能不能开:用「**站长工具 Ping**」这类网站(搜「多地 ping」),输入你的域名,看**国内各地节点**是不是通的。

---

## 常见问题 / 坑

- **打开后无限跳转、打不开** → 99% 是第 4 步的 SSL 模式没设成 **Full**。回去改。
- **一直显示没生效** → DNS 改 Nameserver 后要等,最长 24 小时;Cloudflare 激活邮件到了才算好。
- **两条记录点开是灰色云朵** → 点一下那个云朵图标切成**橙色**。
- **vercel.app 还是打不开** → 正常,那个域名本来就封;以后对外只发**你自己的域名**。
- **后台管理页(/admin)在大陆卡** → 后台是浏览器直连海外 Supabase 导致的,你自己管理时挂梯子即可,不影响普通用户看前台。
