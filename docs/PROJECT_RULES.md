# PROJECT_RULES

## 用途

记录本项目的长期开发规则，确保后续新增页面、后台功能、数据库和部署配置时保持一致。

## 项目定位

- 中文知识技巧、软件工具、AI 工具、开源项目、在线工具分享站。
- 重点是合法的工具发现、知识技巧、软件介绍、AI 工具导航、开源项目介绍。
- 不做盗版资源、破解软件、影视资源、课程资源、电子书资源、网盘资源导流。

## 固定技术栈

- Next.js App Router
- TypeScript
- Tailwind CSS
- Supabase
- Vercel

## 开发规则

- 每次只做一个小步骤。
- 每次改代码前先说明要改哪些文件。
- 每次改完必须说明如何运行和检查。
- 不一次性重构大量代码。
- 不引入没必要的复杂技术。
- 保持代码干净、目录清晰、命名统一。
- 页面必须同时适配电脑端和手机端。
- 后台功能必须简单直观。
- 重要决策要写入文档，避免项目后期熵增。
- 视觉、卡片、封面/媒体资产、真实数据等统一标准见 `docs/DESIGN_SYSTEM.md`（2026-07-09 起为唯一权威版本，之前的浅色方向已作废）；写任何会操作 Supabase 的脚本前先看该文件的"内容资产标准"一节。
- 用户不执行任何命令行/控制台操作，所有实现（含数据库相关操作）由协作者直接完成；确需数据库结构变更（建表/加列）等只能由用户手动在控制台完成的操作，要先与用户确认，并优先寻找 Storage + 脚本的替代方案。

## 命名规则

- 文件夹和文件名优先使用小写英文与短横线，例如 `tool-card.tsx`。
- React 组件使用 PascalCase，例如 `ToolCard`。
- TypeScript 类型使用 PascalCase，例如 `ToolItem`。
- 数据库字段使用 snake_case，例如 `created_at`。

## 变更记录

- 每次完成一个小任务，都要更新 `docs/TASK_LOG.md`。
- 影响项目规则、设计、数据库、内容或后台的改动，要同步更新对应文档。
- 可见版本变化写入 `docs/CHANGELOG.md`。
