# 主页接真实数据 + 复用封面卡片 设计文档

日期:2026-07-09
用户决策:主页接真实数据(而非只填图);主页优先于详情页富媒体
现状:主页 CosmicHomeSections 渲染 cosmic-home-data.ts 的写死演示数据;page.tsx 已 await getPublishedTools()/getPublishedArticles() 但结果被丢弃。空图位:精选大卡 visual(1)、最新指南缩略图(8)。
真实数据:13 个已发布工具(均有封面) + 3 篇已发布文章。

## 目标

主页展示真实收录内容,空图框由真实封面自动填满,版式复用第一期重做的带封面卡片。主页不再是样板间。

## 决策

1. **数据从 page.tsx 传入**:`getPublishedTools()`/`getPublishedArticles()` 结果作为 props 传给 CosmicHomeSections,不再丢弃。
2. **精选区**:hero = 1 个真实工具(优先有官方照片封面的),用 `FeaturedToolCard`;侧栏 = 接下来 4 个工具,用 `CompactToolCard`。复用已建组件,自带封面/图标/兜底链。
3. **最新区**:8 张假指南卡 → 复用 `ArticleCard` 渲染 3 篇真实文章(文章卡本就是纯排版无缩略图,空缩略图问题随之消失)。
4. **榜单区**:榜单的"阅读量 8.2K"是我们不追踪的假数据。改为诚实呈现——按分类分组真实工具的紧凑列表(无假数字),或并入"更多工具"网格。取真实工具按分类聚合,只显示有工具的分类。
5. **分类入口区**:保留(它是场景网关,非工具);count 数字先保留占位或后续接真实计数(低优先)。
6. **合作伙伴区**:保留(广告位,合理留白)。
7. **卡片 CSS 全局可用**:`.zh-tool-card` 等未被 `.zh-tools-page` 限定,可在主页 warm-home 作用域直接使用。

## 分步(小步走,每步截图验证)

- **3-1 精选区接真实工具**(最大空框,先做):page.tsx 传数据 → 精选区换 FeaturedToolCard + CompactToolCard。
- **3-2 最新区接真实文章**:换 ArticleCard × 3。
- **3-3 榜单区接真实工具**:按分类聚合的真实工具列表,去掉假阅读量。
- **3-4 清理**:删除死掉的 zh-feature-visual/zh-guide-thumb 空图 CSS 与不再引用的 home-* 演示卡组件/数据。

## 边界

不碰:/tools 与 /articles 列表页(已定稿)、详情页、后台、数据库结构、分类入口的图标风格。

## 验证与回滚

每步 tsc + 桌面/移动截图;数据为空时各区回落友好空态(不报错)。回滚:各区块改动独立,可单独 revert;演示数据文件保留到 3-4 才删。
