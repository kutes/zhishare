# ROADMAP

## 用途

记录项目阶段目标，防止功能越做越散。

## 当前阶段

第 1 阶段：项目基础建设。

目标：

- 完成项目初始化检查。
- 建立基础文档体系。
- 初始化 Next.js App Router + TypeScript + Tailwind CSS 项目骨架。
- 暂不开发业务功能。

## 后续阶段草案

第 2 阶段：前台基础页面。

- 首页基础布局。
- 工具列表页。
- 工具详情页。
- 分类与标签展示。
- 移动端适配。

第 3 阶段：内容数据与 Supabase。

- 设计数据库表。
- 接入 Supabase。
- 读取公开内容。
- 保持数据库结构简单清晰。

第 4 阶段：简单后台。

- 管理员登录。
- 工具内容新增、编辑、下架。
- 分类与标签管理。
- 内容审核状态。

第 5 阶段：上线与维护。

- Vercel 部署。
- 基础 SEO。
- 性能与可访问性检查。
- 内容合规检查流程。

## 阶段原则

- 每个阶段先完成可运行、可检查的小闭环。
- 不提前加入复杂权限、复杂工作流或复杂推荐算法。
- 任何新增功能都必须服务网站定位。
