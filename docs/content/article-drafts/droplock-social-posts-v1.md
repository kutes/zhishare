## X 长文版

我最近测了一个很小众的工具：DropLock。

它不是 AI 工具，也不是效率神器。
它解决的是一个很土、但很真实的问题：你有没有直接把 API Key、数据库密码、临时邀请码发到微信、QQ、Telegram 或邮件里？

很多小团队都干过这件事。
顺手，但风险很高。

我这次用虚构 secret 跑了一遍 DropLock 的流程：

- 不需要注册账号；
- 可以生成 lock box link；
- 发送方可以输入虚构 secret；
- 可以生成带 `#m=` 的 secret link；
- 网络层没有观察到 POST / PUT / PATCH 携带虚构 secret；
- 请求主要是 GET 文档和 logo；
- 官方页面明确提醒：没有经过安全专家审计。

所以我现在不会把它写成“可以立刻上手的结论”。

我更愿意把它看成一个值得继续观察的本地脱敏工具。

它至少提醒我一件事：

```text
我现在传的不是普通文字，而是 secret。
```

这句话本身，就很值钱。

## 小红书正文短版

我最近试了一个本地秘密分享工具：DropLock。

它的思路很直白：先生成 lock box link，再把 secret 锁进去，最后返回一个带 `#m=` 的链接。

我用的是虚构样本，不是真实账号信息。

观察下来：

- 页面能打开；
- 可以输入 secret；
- 可以生成分享链接；
- 网络里没有看到虚构 secret 通过 POST / PUT / PATCH 直接上传；
- 但官方也明确提示它没有经过安全专家审计。

所以我现在不会把它写成“安全神话”。

它更像是一个提醒你“别把 secret 当普通文字发出去”的工具。

如果你也常在聊天框里发 API Key、密码、临时邀请码，至少先记住一句话：

```text
secret 不是普通文本。
```
