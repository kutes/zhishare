# PlainApp 低敏设备实测前置确认 v1

Status: preflight

## 1. 当前目标

本步骤只判断 PlainApp 是否具备进入低敏实测的前置条件。

本步骤不安装 PlainApp，不连接 Android 手机，不读取真实数据，不开启屏幕镜像，不进行真实测试。

## 2. 为什么必须先做前置确认

PlainApp 不是普通网页工具。

根据已落盘的官方来源备注，PlainApp 可能涉及：

- 文件管理
- 照片
- 视频
- 音乐
- 联系人
- SMS
- 通话记录
- 通知
- 屏幕镜像
- 远程控制
- 设备信息
- 局域网浏览器访问
- HTTPS / 证书
- Android 权限
- 分发渠道版本差异

这些都属于高敏边界。

所以不能拿主力手机直接实测。

## 3. 已确认的官方声明边界

当前只记录官方声明，不作为 zhishare 实测结论。

已确认来源中包含：

- 官方官网
- GitHub 仓库
- AGPL-3.0 许可候选
- privacy policy 候选
- terms 候选
- Docs & FAQ 候选
- Security Deep Dive 候选
- Google Play / F-Droid / GitHub 分发渠道候选

官方声明包含：

- no signup
- no cloud
- open source
- same network browser access
- local phone web portal
- TLS + XChaCha20-Poly1305
- 文件、照片、视频、短信、通话记录、屏幕镜像等功能

注意：

no cloud 不等于没有风险。
同网段访问不等于只有自己能访问。
开源不等于自动安全。
官方加密声明不等于安全审计。

## 4. 进入真测前必须满足的条件

必须满足：

- 用户明确同意
- 使用备用 Android 设备
- 不使用主力手机
- 设备里不能有真实照片
- 设备里不能有真实短信
- 设备里不能有真实联系人
- 设备里不能有真实通话记录
- 设备里不能有真实公司文件
- 设备里不能有真实客户资料
- 只连接可信局域网
- 不在公共 Wi-Fi 测试
- 测试前记录 Android 版本
- 测试前记录安装来源
- 测试前记录 app 请求的权限
- 测试时记录浏览器访问 URL、端口、证书提示
- 测试时记录是否有外部请求
- 测试后确认 Stop 后浏览器是否还能访问
- 测试后卸载或清理测试环境

## 5. 低敏实测样本建议

后续如果进入实测，只能使用：

### 文件样本

- `plainapp-demo-note.txt`
- `plainapp-demo-folder/`

### 图片样本

- 自制纯色测试图片
- 不使用真人照片
- 不使用身份证、护照、合同、票据截图

### 联系人样本

- 张三测试
- 李四 Demo
- 电话使用虚构号码

### 短信样本

- 只使用测试短信
- 不包含验证码
- 不包含真实交易、登录、银行、社交内容

### 通话记录样本

- 只使用测试设备产生的空记录或虚构记录
- 不使用真实亲友号码

## 6. 观察项

如果进入真测，需要记录：

- 是否需要账号
- 是否需要互联网
- 是否只在同网段访问
- 浏览器 URL 和端口
- HTTPS / 证书提示
- app 请求的 Android 权限
- 文件管理是否可用
- 照片 / 视频是否可见
- SMS / 通话记录是否可见
- 屏幕镜像是否可开启
- Stop 后浏览器是否还能访问
- 切换网络后是否无法访问
- 是否出现外部网络请求
- 小白是否能理解风险

## 7. 当前不允许做的事

当前不允许：

- 下载
- 安装
- 启动
- 授权
- 扫描局域网
- 打开真实手机数据
- 改 observed
- 写文章
- 发布文章
