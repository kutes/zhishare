# 免费 AI 工具安全吗？官方来源核验表 v1

## 1. 目标

这份文档用于核验《免费 AI 工具安全吗？新手应该注意什么》这篇文章的官方来源。

本阶段不写最终结论，只建立核验框架。

文章发布前，必须人工打开官方页面核验：

- 官方入口
- 帮助中心
- 价格页
- 隐私政策
- 数据使用说明
- 删除数据 / 数据控制说明
- 商用或使用条款
- 免费额度或订阅说明

## 2. 候选工具

第一轮候选工具：

1. ChatGPT
2. Claude
3. Gemini
4. Perplexity
5. Poe

## 3. 每个工具必须核验的问题

每个工具都要回答：

- 官方入口是否正确？
- 是否需要账号？
- 是否有免费入口？
- 免费额度是否写清楚？
- 是否说明用户输入会不会被用于训练？
- 是否说明数据保留时间？
- 是否可以关闭训练或改善模型用途？
- 是否可以删除历史记录或数据？
- 是否允许上传文件？
- 是否提醒不要上传敏感信息？
- 是否有商用使用说明？
- 是否有地区限制？
- 是否有未成年人或儿童使用限制？
- 是否有团队版 / 企业版不同数据政策？
- 是否有 API 和网页端不同政策？

## 4. 只允许填写的状态

source_status 允许值：

- pending
- checked
- unclear
- unavailable

verification_status 允许值：

- pending
- verified
- needs_review
- not_applicable

首批统一使用 pending。

## 5. 禁止事项

禁止：

- 没看官方页面就写 confirmed
- 没看价格页就写免费
- 没看隐私政策就写安全
- 没看条款就写可商用
- 没看数据控制说明就写不会训练
- 把第三方博客当官方来源
- 把 AI 回答当官方来源
- 把社媒传言当官方来源
- 把旧截图当最新政策

## 6. 文章写作规则

正式文章里必须这样写：

- “我查到的官方说明是……”
- “截至核验日期……”
- “这个地方我还没有完全确认，所以不下结论。”
- “免费入口不等于没有成本。”
- “不建议上传身份证、客户资料、公司机密、未公开代码。”
- “商用前要看对应工具的最新条款。”

如果没有完成官方来源核验，文章不能发布。
