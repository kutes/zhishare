# 首批工具 staged CSV 复核清单 v1

Status: active

## 1. 复核目标

本清单用于复核 `docs/content/bulk-tools-staged-v1.csv`。

任何工具条目进入发布前，都必须先通过本清单。

## 2. 基础字段检查

每一行必须检查：

- title 不为空；
- slug 不为空；
- slug 不重复；
- category 不为空；
- short_description 不为空；
- long_description 不为空；
- official_website 不为空或明确写待核验；
- source_urls 不为空；
- suitable_for 不为空；
- not_suitable_for 不为空；
- caution_notes 不为空；
- publish_ready 默认为 no。

## 3. 来源检查

每一行必须检查：

- 官网是否正确；
- GitHub / 官方仓库是否正确；
- 是否误填搬运站；
- 是否误填假官网；
- 是否只有网盘链接；
- 是否存在破解 / 盗版 / 灰产风险；
- 许可证是否需要复核。

## 4. 文案检查

每一行必须检查：

- 是否是真实写手风；
- 是否有明确场景；
- 是否有适合人群；
- 是否有不适合人群；
- 是否有注意事项；
- 是否避免空泛；
- 是否避免关键字堆砌；
- 是否避免 AI 味太重。

## 5. 亲测与评论检查

允许：

- 本人亲测；
- 团队亲测；
- 站内实测；
- 网友反馈摘选。

但必须检查：

- 亲测是否来自用户或团队真实确认；
- 网友评论是否来自公开来源；
- 是否把网友评论误写成本人亲测；
- 是否编造具体测试细节；
- 是否编造评论来源；
- 是否把少量评论写成全网结论。

## 6. 风险检查

重点检查：

- 文件访问；
- 麦克风；
- 摄像头；
- 录屏；
- 录音；
- 手机权限；
- 局域网访问；
- API Key；
- 客户资料；
- 外部供应商；
- 账号授权；
- 隐私边界；
- 许可证；
- 价格变化。

## 7. 发布前检查

发布前必须确认：

- 人工确认来源；
- 人工确认 slug；
- 人工确认 category；
- 人工确认 tags；
- 人工确认没有禁止内容；
- publish_ready=yes；
- 再进入导入或后台发布。
