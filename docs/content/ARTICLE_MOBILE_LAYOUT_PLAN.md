# 移动端文章详情页排版设计规划

## 1. 设计目标

移动端文章详情页的目标不是展示更多模块，而是让用户愿意连续读下去。

当前文章内容已经从“参数清单风”改成了“创作者分享风”，下一步移动端也要配合这个方向，做成更轻、更顺、更像阅读页的结构。

移动端文章页的核心目标：

- 首屏快速知道这篇文章讲什么。
- 正文阅读不累。
- 广告不打断核心段落。
- 小标题清晰。
- 段落节奏自然。
- 页脚轻量化。
- 不出现桌面端组件直接缩小的感觉。

## 2. 当前问题诊断

从当前移动端截图看，页面有几个明显问题。

第一，正文区域太像一张长卡片。  
这种方式在工具详情页可以接受，但文章详情页更适合沉浸阅读。文章是连续阅读内容，如果每一段都被卡片、广告、分割区打断，会让读者感觉自己在看组件，而不是看文章。

第二，广告位打断节奏。  
文章正文中间连续出现广告，会破坏阅读情绪。广告可以保留，但移动端必须控制数量和位置，不能每隔一小段就插入。

第三，正文宽度偏窄。  
手机端文章正文应该尽量利用屏幕宽度，但不要贴边。左右 18px 到 20px 的 padding 比较合适。当前如果正文卡片内外都留白，就会导致正文区域被压窄。

第四，Footer 太重。  
手机端页脚应该轻量，最多保留品牌说明、快速链接和版权信息。大量链接标签和完整站点说明适合桌面端，不适合手机阅读页。

## 3. 移动端页面结构

建议移动端文章详情页采用下面顺序：

```txt
顶部导航
文章 Hero
正文阅读区
文中广告 1
继续正文
相关阅读
版权提醒
轻量 Footer
```

不要使用过多独立卡片。  
文章正文应该是一条连续阅读流，而不是多个白色卡片拼起来。

## 4. 顶部导航

移动端顶部导航目标是轻，不抢正文。

建议：

- 高度控制在 48px 到 56px。
- 左侧保留品牌 Logo 和“知享”。
- 右侧保留一个“菜单”按钮。
- 不在顶部堆太多导航项。
- 滚动时可固定，也可以不固定；第一阶段不强制做 sticky。

视觉建议：

- 背景白色或半透明白。
- 底部轻微 border。
- 不使用过重阴影。

## 5. 文章 Hero

文章 Hero 是移动端第一屏最重要区域。

建议结构：

```txt
分类标签
标题
摘要
发布时间 / 阅读时间
```

视觉要求：

- Hero 可以保留浅蓝紫渐变背景。
- 标题卡片可以保留，但不要太厚。
- 标题字号建议 26px 到 30px。
- 标题行高建议 1.18 到 1.25。
- 摘要字号建议 15px 到 16px。
- 摘要行高建议 1.7。
- Hero 上下 padding 控制在 36px 到 48px。
- 不要让 Hero 占据接近一整屏。

## 6. 正文阅读区

正文应该更像文章，而不是卡片组件。

建议：

- 移动端正文容器不使用大卡片包裹。
- 可以使用透明背景或非常轻的白色容器。
- 左右 padding：18px 到 20px。
- 正文字号：16px 或 17px。
- 正文行高：1.8 左右。
- 段落间距：18px 到 22px。
- 小标题上方间距：36px 到 44px。
- 小标题下方间距：12px 到 16px。

建议 Tailwind 方向：

```tsx
article: "px-5"
content: "text-[16px] leading-8 text-slate-700"
h2: "mt-10 text-[22px] font-bold leading-snug text-slate-950"
p: "mt-5"
```

## 7. 小标题样式

小标题不要像 Markdown 标题，也不要太像后台字段。

建议：

- H2 字号 21px 到 23px。
- 字重 700。
- 颜色接近 slate-950。
- 不加编号。
- 不加多余图标。
- 不加复杂装饰。
- 可以在左侧加一条极细的青蓝短线，但不要每个标题都很重。

示例：

```txt
先说结论
我怎么分工
Canva 适合什么时候
我最后怎么选
```

这种标题比“第一类 / 第二类 / 第三类”更自然。

## 8. 段落风格

移动端段落要避免太碎。

内容侧规则：

- 一段尽量 2 到 4 句。
- 不要一句话就频繁换段。
- 不要大量列表。
- 不要 FAQ 堆叠。
- 不要代码块、表格、长项目符号。

前端侧规则：

- 段落不缩进。
- 段落之间用 margin 分隔。
- 文字颜色不要纯黑，用 slate-700 或 slate-800。
- 重点句可以单独成段，但不要过多。

## 9. 广告位规则

移动端文章页广告必须克制。

第一阶段建议：

- 正文中最多 1 个文中广告。
- 文章结束后可以 1 个底部广告。
- 不要在每个小节之间插广告。
- 不要连续出现多个广告。
- 广告块高度控制在 96px 到 140px。
- 广告视觉要弱，不要比正文更抢眼。

推荐广告位置：

```txt
Hero 后不要立刻插广告
正文前 30% 不插广告
读完 2 到 3 个小节后插入 1 个文中广告
正文结束后再放 1 个底部广告
```

如果文章较短，只放底部广告。

## 10. 相关阅读

相关阅读不要太重。

建议：

- 文章结尾后显示“继续阅读”。
- 最多显示 2 篇。
- 卡片高度压缩。
- 不使用大图。
- 展示标题、分类、简短摘要即可。

移动端不要做复杂网格，使用纵向列表。

## 11. 版权提醒

版权提醒可以保留，但要轻量。

建议：

- 放在正文结束后。
- 使用浅灰背景小卡片。
- 字号 13px 到 14px。
- 不要比正文更醒目。
- 文案控制在 3 到 5 行。

## 12. Footer 移动端轻量化

当前移动端 Footer 偏重。

建议移动端 Footer 只保留：

```txt
品牌 Logo + 知享
一句站点说明
快速访问：首页 / 工具库 / 文章
版权信息
```

可以隐藏或折叠：

- 资源
- 参与与反馈
- 内容原则标签
- 多列链接

桌面端 Footer 可以保持完整，移动端单独简化。

## 13. 移动端文章页密度建议

推荐参数：

| 区域              | 建议          |
| --------------- | ----------- |
| 页面左右 padding    | 18px - 20px |
| Hero 上下 padding | 36px - 48px |
| 标题字号            | 26px - 30px |
| 标题行高            | 1.18 - 1.25 |
| 摘要字号            | 15px - 16px |
| 正文字号            | 16px - 17px |
| 正文行高            | 1.75 - 1.85 |
| 段落间距            | 18px - 22px |
| H2 字号           | 21px - 23px |
| H2 上方间距         | 36px - 44px |
| H2 下方间距         | 12px - 16px |
| 文中广告数量          | 最多 1 个      |
| 底部广告数量          | 最多 1 个      |
| 相关阅读数量          | 最多 2 篇      |

## 14. 后续代码改造优先级

后续实际改代码时，建议按这个顺序：

### 第一优先级：正文阅读区

- 去掉移动端正文大卡片感。
- 放大正文。
- 增加行高。
- 控制左右 padding。
- 优化 H2 和段落间距。

### 第二优先级：广告位

- 移动端正文中最多保留 1 个文中广告。
- 底部最多 1 个广告。
- 删除过多广告插入点。

### 第三优先级：Footer

- 移动端 Footer 改为轻量版本。
- 隐藏过多链接分组。
- 保留品牌和关键入口。

### 第四优先级：文章列表页

- 收紧 `/articles` 首屏高度。
- 缩短筛选区高度。
- 让文章卡片更早出现。

## 15. 移动端验收标准

改造完成后，移动端文章详情页必须满足：

- 第一屏能看到标题和摘要。
- 正文宽度舒服，不显窄。
- 字体不小。
- 小标题清楚。
- 段落不碎。
- 广告不连续打断。
- Footer 不压迫阅读。
- 页面不像 Markdown 文档。
- 页面不像后台说明书。
- 页面更像创作者写给普通用户的文章。
