### summary

Figma 更适合做网页、App、产品界面和团队协作设计，不适合只想快速做一张海报或封面的纯小白用户。

### description

Figma 是一个在线设计协作工具，很多人第一次听到它，会以为它只是“高级版画图软件”。但真正用下来，它更像是产品团队一起画界面、改原型、交接开发的工作台。

如果你只是想做一张小红书封面、课程海报，Figma 不一定是最快的。Canva 这种模板工具会更省事。可如果你要做网站页面、App 界面、后台系统、产品原型，或者想让设计稿和开发之间少一点来回沟通，Figma 就更合适。

我觉得新手最容易误会 Figma 的地方是：一上来就想学完整设计系统。其实没必要。刚开始只要会建 Frame、放文字、放按钮、调间距、导出图片，就已经能解决很多实际问题。

### pros

在线协作很方便，别人打开链接就能看设计稿
适合做网页、App、产品原型，不只是普通图片
组件和样式复用能力强，适合长期项目
设计稿可以更清楚地交给开发理解
免费版已经能满足很多个人学习和小项目需求

### cons

刚打开时会比 Canva 更有学习门槛
不适合只想快速套模板做封面的人
如果没有设计基础，很容易把页面做得很乱
多人团队和高级功能可能涉及付费限制
国内访问和协作体验可能受网络环境影响

### risk_notice

Figma 适合做界面和原型，但不要把它当成万能设计工具。新手刚开始不要急着研究设计系统、变量、复杂组件和插件，先学会画清楚一个页面更重要。

如果你使用社区模板、插件、图标包或别人的 UI Kit，要注意授权范围。个人练习问题通常不大，但商业项目、客户项目和公开发布前，最好确认素材能不能商用。
