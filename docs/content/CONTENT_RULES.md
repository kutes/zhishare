# 内容建设规则

本文件是网站内容建设的核心规则。后续工具页、文章页、社交媒体推广文案都必须遵守本规则。

## 1. 内容定位

本网站不是传统百科站，也不是单纯搬运工具介绍的网站。

本网站内容核心是：

> 我试过 - 哪里坑 - 怎么选

所有内容必须帮助中文小白用户快速判断：

- 这个工具到底能不能解决我的问题？
- 它适合我吗？
- 我会卡在哪里？
- 有没有隐私、版权、付费、下载风险？
- 我现在应该怎么选？

## 2. GEO 内容原则

GEO 指面向 AI 搜索、AI 摘要、AI 推荐环境的内容组织方式。

内容必须方便人读，也必须方便 AI 抓重点。

每篇内容都应该做到：

1. 先给结论。
2. 再说明原因。
3. 再给具体例子。
4. 最后给行动建议。
5. 使用清晰标题、短段落、列表、表格、FAQ。
6. 避免空泛形容词。
7. 避免只写“是什么”。
8. 必须写“适合谁 / 不适合谁 / 风险提醒”。

## 3. 禁止的内容风格

禁止使用报告腔、百科腔、营销空话。

不要写：

- 随着人工智能时代的到来……
- 在数字化浪潮之下……
- 本文将深入探讨……
- 这是一款功能强大的工具……
- 它可以极大提升效率……
- 堪称神器……
- 必备工具……
- 全网最强……
- 零基础也能秒变高手……

## 4. 推荐的内容开头

推荐使用体验型、判断型、避坑型开头。

示例：

- 我试了 3 个同类工具，真正适合新手的只有 1 个。
- 这个工具看起来很强，但小白最容易卡在 3 个地方。
- 如果你只是想快速出结果，不建议一上来就用最复杂的那个。
- 这个工具最大的优点不是功能多，而是能让新手少走弯路。
- 我不建议所有人都用它，尤其是只想临时完成小任务的人。

## 5. 工具内容必须回答的问题

每个工具页至少要回答：

1. 这个工具一句话是什么？
2. 它真正解决什么问题？
3. 适合谁？
4. 不适合谁？
5. 小白第一次用会卡在哪里？
6. 免费版够不够？
7. 是否需要注册？
8. 有没有隐私风险？
9. 有没有版权风险？
10. 官网入口是否可靠？如果正文提到下载方式，是否明确建议走官网？
11. 有没有替代工具？
12. 最后建议怎么选？

## 6. 文章内容必须回答的问题

每篇文章至少要回答：

1. 这篇文章给谁看？
2. 读完能解决什么问题？
3. 小白最容易误解什么？
4. 哪些地方容易踩坑？
5. 应该怎么判断？
6. 有哪些具体例子？
7. 最后应该怎么做？

## 7. 内容语气

默认面向中文小白用户。

语气要求：

- 直接
- 清楚
- 有判断
- 有体验感
- 不装专业
- 不吓唬用户
- 不承诺结果
- 不强行制造焦虑

可以指出风险，但不能夸大风险。

## 8. 工具页字段使用原则

当前工具表已有字段：

- title / name
- slug
- summary
- description
- website_url
- download_url
- cover_url
- category_id
- is_free
- is_open_source
- target_users
- use_cases
- pros
- cons
- risk_notice
- status

第一阶段优先复用现有字段，不新增数据库字段。

字段写法建议：

| 字段 | 内容用途 |
|---|---|
| summary | 一句话结论 |
| description | 工具是什么 + 真正解决什么问题 |
| target_users | 适合谁 |
| use_cases | 使用场景 / 新手可完成的任务 |
| pros | 优点 |
| cons | 缺点 / 不适合谁的一部分 |
| risk_notice | 风险提醒 / 坑点 |
| website_url | 官网 |
| download_url | 第一阶段强制留空，不填写网盘、第三方下载站或安装包直链 |

第一阶段额外规则：

- `download_url` 必须留空。
- 即使工具有桌面端、安装包或客户端，当前阶段也只保留 `website_url` 作为官网入口。
- 不把网盘、镜像站、第三方下载站、安装包直链写进 `download_url`。

## 9. 当前暂不强行实现的内容模块

以下模块目前没有专用工具字段，暂时不要强行扩数据库：

- 我试过什么
- 不适合谁
- 怎么选
- 新手步骤
- FAQ
- 最后建议

这些内容可以先放在文章系统里承接，或者后续统一设计字段后再做。

## 10. 风险提醒规则

涉及以下内容时，必须写风险提醒：

- 第三方下载站 / 网盘资源 / 非官网安装包
- 软件安装包
- 破解资源
- 设计素材
- 字体
- 图片
- 视频素材
- AI 生成内容
- 账号登录
- 文件上传
- 隐私数据
- 付费订阅
- 自动续费
- 开源项目本地部署

风险提醒不能只写“注意风险”，必须写具体风险。

示例：

- 如果你要上传身份证、合同、客户资料，不建议直接使用陌生在线工具。
- 如果正文提到下载方式，建议明确提示用户优先从官网进入，避免下载到捆绑安装包或二次打包文件。
- AI 生成图片不等于一定可商用，商用前要看平台授权条款。
- 免费版通常会有限制，正式使用前要确认导出、水印、次数和商用权限。

## 11. 标题规则

标题要像真实用户会搜索的问题，但要带判断和体验。

不推荐：

- Canva 工具介绍
- ChatGPT 是什么
- Notion 使用教程
- AI 工具推荐

推荐：

- 我试了 Canva，发现它最适合这 3 类设计小白
- ChatGPT 新手最容易用错的地方，不是提示词太短
- Notion 看起来很强，但不是所有人都适合用
- 免费 AI 工具一定安全吗？我建议先看这 4 点

## 12. 内容发布前最低标准

每篇内容发布前必须通过检查：

- 是否先给结论？
- 是否有明确目标用户？
- 是否说明适合谁？
- 是否说明不适合谁？
- 是否指出至少一个坑点？
- 是否有具体使用场景？
- 是否有风险提醒？
- 是否避免百科腔？
- 是否避免夸大承诺？
- 是否方便 AI 抓取重点？
