# 内容发布检查清单

每篇工具内容或文章发布前，都必须按本清单检查。

## 1. 基础检查

- [ ] 标题是否清楚？
- [ ] 是否先给结论？
- [ ] 是否说明目标用户？
- [ ] 是否避免百科腔？
- [ ] 是否避免报告腔？
- [ ] 是否没有夸大承诺？
- [ ] 是否没有虚假体验？
- [ ] 是否没有“全网最强”“必备神器”等绝对化说法？

## 2. GEO 检查

- [ ] 是否方便 AI 抓取重点？
- [ ] 是否有清晰小标题？
- [ ] 是否有短段落？
- [ ] 是否有列表？
- [ ] 是否能回答用户怎么选？
- [ ] 是否有适合谁？
- [ ] 是否有不适合谁？
- [ ] 是否有风险提醒？
- [ ] 是否有行动建议？

## 3. 工具页检查

- [ ] summary 是否是一句话结论？
- [ ] description 是否说明真正解决的问题？
- [ ] target_users 是否写了具体人群？
- [ ] use_cases 是否写了具体场景？
- [ ] pros 是否具体？
- [ ] cons 是否具体？
- [ ] risk_notice 是否具体？
- [ ] website_url 是否为官网或可信入口？
- [ ] download_url 是否保持为空？
- [ ] 是否没有填写网盘、第三方下载站、镜像站或安装包直链？
- [ ] 如果正文提到下载方式，是否明确建议从官网进入？

## 4. 文章检查

- [ ] 开头是否直接给结论？
- [ ] 是否写清楚测试范围或观察范围？
- [ ] 是否没有伪装成全网权威？
- [ ] 是否有具体例子？
- [ ] 是否有对比或判断标准？
- [ ] 是否有小白可执行步骤？
- [ ] 是否有 FAQ 或常见疑问？
- [ ] 是否有最后建议？

## 5. 版权与合规检查

- [ ] 是否没有提供破解软件？
- [ ] 是否没有引导规避付费？
- [ ] 是否没有搬运他人完整文章？
- [ ] 是否没有未经授权使用他人图片？
- [ ] 是否没有暗示网盘资源一定安全？
- [ ] 是否没有把非官方链接写成官网？
- [ ] 是否提醒用户确认商用授权？
- [ ] 是否提醒用户注意隐私文件上传风险？

## 6. 发布前最后检查

- [ ] 中文是否通顺？
- [ ] 是否有错别字？
- [ ] 是否有乱码？
- [ ] 是否有连续问号？
- [ ] 是否适合手机阅读？
- [ ] 是否段落过长？
- [ ] 是否可以拆成列表或表格？
