[速览]
- Bento 是一个「幻灯片工具」，但整个应用——查看器、播放器、编辑器——全部塞进了**一个 HTML 文件**里（约 560 KB）。你把这个文件发给别人，对方什么都不用装，用浏览器打开就能看、能编、能放映。
- 它的核心理念是「文件即软件」：幻灯片数据以一段**明文 JSON** 存在文件顶部，人能读、AI 能改；保存时文件把新数据写回自己（用浏览器的文件系统 API）。
- 主打本地优先：开启离线模式后「什么都不出本机」，联网和协作会被硬性阻断。它也带可选的实时协作——端到端加密（AES-GCM），密钥只存在你的文件里、不上服务器，中继只能看到密文。
- 因为文档就是一段明文 JSON，它对 AI 特别友好：Claude Code、Cursor 等能直接改这个 `.bento.html`；接本地开源模型（Ollama 等）离线改也行。开源（MIT），GitHub 590+ 星。
- 冷静一句：这是个**刚出生一周的新项目**（仓库 2026-07-17 创建），作者账号也是新号、暂无维护历史——概念惊艳、值得玩，但要托付重要演示前，请把它当「早期项目」看待。

## [事件] 一个 HTML 文件，装下了整个「PPT 软件」

我们对「文档」的默认想象是：文档是内容，软件是另一装的东西——你得先有 PowerPoint，才能打开 .pptx。Bento 把这件事反过来了：**软件本身就住在文档里**。

一个 Bento 幻灯片，就是一个约 560 KB 的 `.bento.html` 文件。这个文件里同时打包了：幻灯片数据、字体、图片、图表、动画，以及一整套**查看器 + 放映器 + 编辑器**。你用任何现代浏览器打开它，它就是编辑器；改完点保存，它会把新数据重写回自己这个文件。你把文件发给别人，对方**什么都不用装、不用注册、不用联网**——「你发过去的这个文件，本身就是软件」。想 10 秒试一下，直接打开 bento.page/slides 就是完整的应用跑在一个起始 deck 上。

[WHY] 为什么值得看：它戳中了一个越来越普遍的不适——办公文档正从「你拥有的东西」变成「你租用的东西」，锁在某家公司的云上、某个登录后面，只有在人家服务器还开着的时候才读得到。Bento 走了另一条路：**一个文件，长期自持**。2026 年存的一份 deck，理论上 2036 年用浏览器还能原样打开。无论你用不用它，这都是对「单文件 Web 应用」这个思路的一次相当完整的演示。

## [能力] 它不只是「能显示」，是真能编能演

单文件不代表功能简陋。Bento 在那个 560 KB 的壳子里塞进了相当完整的一套东西。

[KEY 一个文件里的完整功能] **编辑与放映**：多幻灯片编辑、演讲者视图、备注、评论、版式、PDF 导出、8 种界面语言。**Morph 动画**：不同幻灯片上共享同一 id 的元素，会在切页时自动补间（位置、大小、颜色乃至渐变都能动）——复制一页、挪动一下，动效自己就生成了。**内置图表**：柱/线/饼/散点由其自研、无外部依赖的引擎绘制，放映时还能实时交互（提示、缩放，甚至柱状图变饼图时数据会「morph」过去）。这些能力，官方用一句话概括：**「装进一个文件里的办公套件」**。

值得留意的是它目前的边界：Bento 定位是一个叫「Bento/Suite」的套件，但**目前只有 Slides（幻灯片）这一个应用真正上线**，官网上 Docs（文档）和 Sheets（表格）都还标着「coming soon」。

## [AI] 为什么说它「为 AI 而生」

Bento 有一个设计上的巧思，让它天然适配 AI 工作流。

[KEY 文档就是一段明文 JSON] 因为幻灯片数据是文件顶部一段**明文、可读的 JSON**（不是二进制格式），任何能读写文件的助手都能直接改你的 deck——**不用插件、不用 API**。两条路子：一是**文件型工具**（Claude Code、Cursor、Aider 等有文件系统权限的 agent）直接就地改那段 JSON，Claude Code 甚至有打包好的 `bento-slides` 技能；二是**任意聊天机器人**——把文档 JSON 复制出来、让它改写、再粘回去。官方还强调，这套流程**接本地开源模型（Ollama、llama.cpp、LM Studio）能完全离线跑，什么都不出本机**。

这也解释了为什么「单文件 + 明文 JSON」的组合在当下突然有意义：在 AI 能可靠读写结构化文本的今天，一个人人和机器都能直接读改的文档格式，比锁在二进制里的传统格式好用得多。

## [冷静] 惊艳，但请按「早期项目」对待

概念很漂亮，实测也确实能跑，但边界同样要说清楚。

[KEY 三条提醒] **第一，它非常新**：GitHub 仓库 2026-07-17 才创建，作者也是新账号、暂无可查的维护/安全历史——这一点在 Hacker News 讨论里就有人当面提出。惊艳归惊艳，重要演示别只存这一份、别只依赖它。**第二，协作和「自我保存」有前提**：保存靠浏览器的文件系统 API（不支持时退化为下载另存）；实时协作虽是端到端加密、密钥只在文件里，但「谁拿到文件谁就是成员」，分享时要意识到「文件 = 通行证」，重要 deck 记得用它的「轮换密钥」来撤销访问。**第三，功能虽全但仍在早期**：HN 里已有用户反馈平板上演讲者备注、放映按钮等交互还不顺手——它是为桌面多屏场景先做的。

把这些放在一起看：Bento 不一定马上取代你的 PowerPoint，但它把「文档能不能不依赖云和账号而长期自持」这个问题，做成了一个你现在就能亲手打开、亲手编辑的实物。光这一点，就值得点开 bento.page/slides 玩十秒钟。
