[速览]
- 传文件卡壳时先分清需求:是"把这几个文件发过去一次就完事",还是"两台设备的文件夹一直保持一致"。
- 一次性发送用 LocalSend:同一 Wi-Fi 下打开就互相看见,像 AirDrop,不用账号也不用数据线。
- 持续同步用 Syncthing:在自己的多台设备间自动同步文件夹,像一个私有网盘,数据不经过任何服务器。
- 两个都是开源、点对点直传,文件不上传到别人的服务器——顺带把网盘最烦的来源和隐私问题绕开了。

[来源] 本文基于两款开源工具的实际使用与官方公开说明整理,支持平台与功能可能随版本更新变化,以 LocalSend 与 Syncthing 官网及 GitHub 项目页为准。

## [先分清] 先分清你要"发一次"还是"一直同步"

传文件不顺手时,大多数人第一反应是打开网盘上传、再把链接发过去——但很多场景根本用不着绕这一圈。

真正该先问的是:你要的是"把这几个文件发过去,发完就完事",还是"这台电脑改了,那台手机自动跟着变"。前者是一次性发送,后者是持续同步,是两种完全不同的活,对应两个不同的工具,用错了会别扭。

[WHY] 网盘把简单的传文件变复杂了:要上传、要生成链接、文件还在别人服务器上转一圈。先认清自己是"发一次"还是"一直同步",就能跳过网盘,直接用更对口、更快、也更私密的工具。

## [LocalSend] 发一次:同一 Wi-Fi 下,像 AirDrop

要把手机里的照片、一个安装包、一段文字发到电脑,LocalSend 是最省事的路径。

两台设备都装上、连着同一个 Wi-Fi,打开 App,对方的设备名就自动出现在列表里——点设备、选文件、对方确认接收,传输就完成了,第一次用基本不用看教程。数据走局域网直传,不经过任何服务器,也不用注册账号,Windows、macOS、Linux、Android、iOS 全平台都能装。

[KEY 适用 · LocalSend] **同一局域网内的一次性传输**选它,像跨平台版的 AirDrop;要注意它必须在同一个 Wi-Fi 下,跨网络远程传不了,公司或公共 Wi-Fi 做了设备隔离时也可能互相看不见。

## [Syncthing] 一直同步:多设备保持一致,像私有网盘

如果需求是"这台电脑的文件夹,和手机、和另一台电脑一直保持一样",那是同步,不是发送,该用 Syncthing。

在每台设备上各装一份,首次配置时两台设备互相加一次"设备 ID"(扫码或复制粘贴),再挑好要同步的文件夹——之后任何一边的改动都会自动加密直传到另一边,不经过第三方服务器,同步多少东西只看你自己的硬盘。它是同类开源工具里成熟度和口碑最高的之一,适合拿来做私人文件同步和多设备备份。

[KEY 适用 · Syncthing] **多设备之间的持续自动同步**选它,相当于自己搭一个不经过云的私有网盘;两点要适应:设备都在线时才会同步(没有"云端常驻副本"),iOS 没有官方客户端要靠第三方应用。

## [怎么选] 一句话分工,外加一个隐私彩蛋

分工其实很干脆:发完就不用管的,LocalSend;要长期保持一致的,Syncthing。两者不冲突,常见搭配是平时用 LocalSend 随手发,重要资料用 Syncthing 常态同步。

还有一层顺带的好处,正好接上"下载资源先看来源"那件事:这两个工具都是点对点直传,文件不落到任何第三方服务器上,来源清楚、隐私也留在自己手里。比起把文件传上网盘再分享,少了一道"内容在别人服务器上转一圈"的风险。
