# 内容采集与二创流水线计划

## 目标

把外部公开资源站的新增内容，按天自动汇总到飞书多维表格或 Notion 数据库，先做合规审查和 AI 初筛，再进入人工复核和二创发布。

## 核心原则

1. 只抓取公开可访问内容。
2. 只保存结构化事实信息，不复制原文段落、图片和版式。
3. 先筛选，再二创，再发布。
4. 所有高风险内容都必须人工复核。
5. 自动化只负责收集和打分，不直接发布到 zhishare。

## 推荐技术栈

- 采集：Python
- 调度：n8n 或 Make.com
- 清洗：Python + BeautifulSoup
- 动态页面：Playwright，必要时再补
- 存储：飞书多维表格或 Notion 数据库
- AI 初筛：模型打分 + 规则过滤

## 字段设计

最小字段：

- 标题
- 链接
- GitHub 星数
- 简介
- 发布时间
- 关键词

建议扩展字段：

- 来源站点
- 分类
- 是否开源
- 是否免费
- 原始摘要
- AI 分数
- AI 结论
- 审核状态
- 是否已二创
- 备注

## 采集顺序

1. RSS
2. 站内分类页
3. GitHub 项目页
4. 文章详情页

优先级原因：

- RSS 最稳定，增量最清楚。
- 分类页便于补充热点。
- GitHub 页适合抓星数和开源属性。
- 详情页只在需要补充摘要时使用。

## AI 初筛维度

建议先按 0 到 100 分打分，再决定是否进入二创池。

- 热度
- 实用性
- 创新度
- 是否开源
- 中文读者适配度
- 二创价值

建议结论：

- 80 分以上：优先二创
- 60 到 79 分：待人工复核
- 60 分以下：先留档，不进入二创

## 合规审查规则

直接剔除：

- 破解、盗版、灰产、规避限制类内容
- 需要登录后才能访问且无明确授权的内容
- 明显侵权的图片、视频、文章片段
- 带有高风险下载链路或诱导性安装包的内容

谨慎处理：

- 带推广性质的资源
- 需要付费或试用门槛的内容
- 涉及隐私、账号、文件上传的工具
- 可能存在版权争议的素材站

## 二创输出要求

发布到 zhishare 前，建议统一成以下结构：

1. 一句话结论
2. 这个内容解决什么问题
3. 为什么值得看
4. 适合谁 / 不适合谁
5. 值得注意的坑点
6. 关键词和外链

表达方式要保持原创，不照搬来源站的标题节奏和描述句式。

## 建议实施阶段

### 第 1 阶段：打底

- 确认采集源清单
- 确认数据库字段
- 确认去重规则
- 确认人工审核入口

### 第 2 阶段：自动采集

- 实现 Python 抓取脚本
- 每天定时抓取新增内容
- 写入飞书多维表格或 Notion

### 第 3 阶段：AI 预筛

- 为每条内容输出分数
- 标记“值得二创 / 待复核 / 先留档”
- 把低分内容自动降噪

### 第 4 阶段：二创工作流

- 人工确认候选内容
- 按 zhishare 内容规范重写
- 发布到对应栏目

## 当前建议

先做最小闭环：

1. 选 3 到 5 个稳定来源
2. 只抓标题、链接、简介、发布时间、关键词、GitHub 星数
3. 先写入飞书或 Notion
4. 再加 AI 打分
5. 最后再接人工复核和发布
